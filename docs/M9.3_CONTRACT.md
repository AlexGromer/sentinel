## Контракт control-API (фаза-1 — следующий под-шаг)

Тонкий **Go HTTP-сервис** (`cmd/control-api`, stdlib `net/http`, как M8 report-service — без новых зависимостей).
Это «не-MCP» путь; MCP-путь (M7) сосуществует (ADR-023).

**Эндпоинты (v1):**
| Метод | Путь | Назначение |
|---|---|---|
| GET | `/healthz` | liveness (без auth) → `{status:"ok", version}` |
| GET | `/v1/config-schema` | поверхность RunConfig (для WebUI) — ключи/дефолты из `runconfig.py`/флагов |
| POST | `/v1/runs` | старт прогона из `{target, runConfig|mode/goal/describe/planner/budgets/auth, env?}` → `{run_id}` (спавнит `agentctl run …`, как orchestrator) |
| GET | `/v1/runs/{id}` | статус: `{state: running|done|failed, exit_code, artifact_dir}` |
| GET | `/v1/runs` | список последних прогонов |

**Безопасность (ADR-032):**
- **Bind на `127.0.0.1` по умолчанию** (`CONTROL_API_ADDR`, не `0.0.0.0` без явного согласия) — спавн
  прогонов = чувствительная поверхность (RCE-класс при публичной экспозиции).
- **Bearer-token** (`CONTROL_API_TOKEN`) обязателен для мутаций (`POST /v1/runs`); `/healthz` — открыт.
  Без токена в env — мутации **403** (read-only/health only).
- **CORS-allowlist** (`CONTROL_API_CORS_ORIGINS`, напр. `https://alexgromer.github.io,http://localhost:8088`):
  ⇒ **Pages-страница может драйвить ЛОКАЛЬНЫЙ control-API** (localhost — исключение mixed-content + CORS).
  По умолчанию — пусто (CORS off; только same-origin/бандл).
- Спавн только через известный путь `agentctl` + валидация входа (никаких произвольных команд).

## setup-WebUI фаза-2 (с control-API)
- Поле **«control-API URL»** (по умолчанию пусто = генерация-онли; `http://localhost:PORT` = живой режим).
- Если задан + `/healthz` ок → разблокировать **«▶ Запустить»** (POST `/v1/runs` с собранным конфигом) +
  опрос **статуса** (GET `/v1/runs/{id}`). Токен — поле (хранится только в памяти вкладки, не в конфиг/URL).
- **Тот же файл** работает: с Pages (→ localhost), из бандла (`--profile webui`), офлайн (генерация).

## CI-шаблоны (GAP-M9-12)
Sentinel = CLI + структурные exit-коды 0/1/2/3 ⇒ заходит в любой CI на коммит. Дать готовые шаблоны
**для пользователя** (не наш GitHub Actions): `Jenkinsfile`, `.gitlab-ci.yml` (replay-гейт по exit-коду).
Это шаблоны в `docs/ci-templates/` или `examples/` — **не** трогают `.github/workflows/` (территория CI-PR #2).
