## Область

Две независимые pw-executor-возможности из M9 design freeze, доставленные одной волной (общий файл
`pw-executor/src/server.ts`, поэтому стек поверх PR #3):

- **M9.4 / A5 — perception in-app вкладок** (`role=tab`): tab-виджеты теперь видимы как интерактивные.
- **M9.4 / A6 — браузерные вкладки/окна** (multi-page): pw-executor отслеживает popups/new-tab и умеет
  переключать активную вкладку.
- **M9.5 / §I — инъекция `traceparent`** во все запросы браузера: каждое UI-действие маппится на
  сквозной backend-trace заказчика (если его сервисы OTel-инструментированы).

## Контракт (что реализовано)

### A5 — perception вкладок
- `browser.interactives` и `browser.setOfMarks` теперь собирают селектор
  `… , [role=button], [role=tab]` → tab-элементы попадают в карту как кликабельные. Клик по табу =
  навигация-внутри-страницы (контент `tabpanel` меняется → re-perceive существующим snapshot).
- **Без нового кода в brain** — tabs текут через уже существующую карту interactives/site-map.

### A6 — мульти-страница
- pw-executor держит `pages: Page[]` (initial + popups/new-tab через `context.on('page')`); `page` —
  **активная** вкладка. Новые инструменты:
  - `browser.tabs` → `{ tabs: [{index, url, title, active}] }` (закрытые вкладки отфильтрованы).
  - `browser.switchTab` `{index}` → делает вкладку активной (`page = pages[index]` + `bringToFront`).
- **Ключевое свойство**: переключение лишь переустанавливает глобал `page` — **все существующие
  инструменты** (`navigate`/`click`/`fill`/`snapshot`/…) работают на активной вкладке без изменений.
- Оба инструмента добавлены в `TOOL_METHODS` (оба транспорта: JSON-RPC + MCP).

### M9.5 — `traceparent`
- В `ensureBrowser`, **если задан `OTEL_EXPORTER_OTLP_ENDPOINT`** (трейсинг включён), регистрируется
  `context.route('**/*')`, добавляющий заголовок `traceparent` (из активного OTel-спана, через
  `otel.ts::currentTraceparent()`) в каждый исходящий запрос браузера. Без коллектора — **no-op**
  (нет route-перехвата, нулевой overhead).
- Это закрывает §I: UI→frontend→сервис→Kafka→БД сшивается в Tempo/Jaeger **заказчика** (его сторона).
  Sentinel остаётся black-box — **никакого backend-коннектора** (см. DISTRIBUTION §интеграция).
