# M9.1 Contract — "Form/Login/Validation primitives" (frozen 2026-06-26)

> 🌐 [Русский](M9.1_CONTRACT.md) (основная версия) · **English**

First sub-milestone of epic **M9** (`docs/M9_CONTRACT.md`). Goal — the **minimum to live-test real UIs**:
forms, search, **login**, multi-step business processes, and **negative testing** (field validation).
Closes Blocker #1 (M9_CONTRACT §A1) in three parts:

- **GAP-M9-01** — `fill`/`type`/`press`/`select` in `pw-executor` (text entry + keys + `<select>`).
- **GAP-M9-04** — auth: `storageState` precondition + **login-as-test**; creds via env, **never in artifacts**.
- **GAP-M9-10** — assertion layer (`assert` primitive per step) + invalid-input generator (**sketch**; full in M9.2).

Verification principle: **offline-first** — `npx tsc --noEmit` (TS) + a FakeEx suite (Python, no live
browser/network) + `go build ./...`. A live run against a real UI is a **separate step, on explicit "go"**.
GoalPlanner / NL authoring is **M9.2** (the explorer is untouched); new step kinds arrive from
**frozen/hand-authored plans via replay** (and forward-compat in `act`).

## §1. New `pw-executor` tools (wire schemas)

Registered in **three places** (`pw-executor/src/server.ts`): the `dispatchInner` switch, the
`TOOL_METHODS` array, and the MCP `schemas` (zod) map — otherwise MCP `registerTool` gets `inputSchema: undefined`.

| Tool | Params | Playwright (pin `^1.61`) | Result |
|------|--------|--------------------------|--------|
| `browser.fill` | `{locator, value? , secretRef?}` | `loc.first().fill(v,{timeout:5000})` | `{filled:true}` |
| `browser.type` | `{locator, text, clear?}` | `clear`→`fill('')`; then `loc.pressSequentially(text,{timeout})` | `{typed:true}` |
| `browser.press` | `{locator?, key}` | `loc.press(key)` or `page.keyboard.press(key)` | `{pressed:key}` |
| `browser.select` | `{locator, value}` | `loc.selectOption(value)` | `{selected:[…]}` |
| `browser.expect` | `{locator?, condition, expected?}` | base waits (`waitFor`/`waitForURL`) — **non-throwing** | `{ok:bool, actual?}` |
| `browser.saveStorageState` | `{path}` | `context.storageState({path})` | `{path}` |

Notes:
- `type` uses **`pressSequentially`** (`locator.type()` has been **deprecated** since PW 1.38).
- `type` does **not** clear the field (append); `fill` auto-clears — use `fill` for login.
- `press` without a `locator` (`page.keyboard`) needs prior focus; with a `locator` it auto-focuses via actionability.
- `select.value` — string/`{label}`/`{index}`/array (like `selectOption`); `{selected}` is a **result only**, not part of the step.

### `browser.expect` — conditions

`condition ∈ {visible, hidden, enabled, disabled, value_equals, text_contains, count_equals, url_contains}`.
Built on **base `playwright` waiting primitives** (`locator.waitFor({state})`, `page.waitForURL`) in
`try/catch` → `{ok:false}` instead of throwing. **Auto-retry** removes the post-submit navigation race
(`@playwright/test` was **considered and rejected** — GAP-ARCH-001: keep `pw-executor` thin, no new deps).
`actual` is restricted to counts/URL/booleans — **never** `inputValue()` (so it can't echo a secret).

## §2. Plan-step schema (extension)

A step today: `{step_id, action_type, semantic_id, intent, target, locator, alternatives, is_milestone}`.
`action_type` grows from `{navigate, click}` to `{navigate, click, fill, type, press, select, assert}`.

| `action_type` | Extra step fields |
|---------------|-------------------|
| `fill` | `locator`, `value?` **or** `secretRef?`, `alternatives?` |
| `type` | `locator`, `text`, `clear?`, `alternatives?` |
| `press` | `locator?`, `key` |
| `select` | `locator`, `value`, `alternatives?` |
| `assert` | `locator?`, `condition`, `expected?`, `expect_ok?` (default `true`) |

`canonical_plan_hash` hashes the **whole** step dict → any new field enters the hash. Old plans
(navigate/click) are unchanged. **Replay MUST read the step read-only** (the `secretRef` resolution and the
`selectOption` result are written only to `rec`/result, never to the step) — otherwise plan_hash drifts.

## §3. Secrets — `secretRef` + tracing gate (the security boundary)

**Rule:** credentials never reach **our artifacts** (`plan.json`, `llm-transcript.jsonl`,
`heal-report.json`, `rec`, OTel attributes, stderr) **nor** the Playwright `trace.zip`.

1. **`secretRef` instead of `value`.** A secret `fill` step stores `secretRef` — the **env var NAME**
   (not the secret), e.g. `"LOGIN_PASSWORD"`. The value is resolved **only inside `pw-executor`**
   (`process.env[secretRef]`) and never returned. The brain never sees the secret; a plan_hash over
   `secretRef` is safe. If the env is unset → **throw `"secret <REF> not set"`** (the name is not secret),
   **never** `fill("undefined")`.
2. **Log/exception redaction.** Log: `fill <locator> = <redacted>`. A secret `fill` is wrapped in
   `try/catch` → rethrow `"browser.fill failed (secret redacted)"` (closes the leak path where an exception
   `message` flows via `executor.py` → stderr/OTel status).
3. **Tracing gate (the real `trace.zip` leak fix).** The Playwright trace records **DOM snapshots**
   (`input.value`, including `type=password`) **and the network POST body** (Keycloak `username`/`password`
   form). "Pause around the fill" does **not** help (the secret leaves via the submit POST and the DOM
   snapshot; Playwright has no pause/mask API). The only sound minimum: **`context.tracing.start` runs only
   when `PW_NO_TRACE != "1"`**. The login-as-test run sets `PW_NO_TRACE=1` — its sole product is the
   `storageState` file, no trace needed. Prod runs use `STORAGE_STATE` (the password is never typed) → their
   traces are clean by construction. `browser.traceStop` is a no-op if tracing never started. See **GAP-RISK-010**.
4. **Fail-closed (not just operator discipline).** `browser.fill` **throws** if tracing is active (`tracingStarted`)
   — **before** reading `process.env[secretRef]` (the secret isn't even loaded). The brain additionally aborts the
   run (**exit 3**) if a plan contains a `secretRef` step while `PW_NO_TRACE != "1"`. So the absolute invariant is
   held by code, not just operator discipline. **storageState load**: the file is parsed inside pw-executor (not
   passed as a string path to `newContext`), so a **corrupt/empty** file falls back to no-state (we log only the
   path, never the bytes). `browser.expect`'s inner waits are bounded by `page.setDefaultTimeout(5000)` — a fast
   `{ok:false}` for negative tests instead of Playwright's 30s default.

## §4. Auth — `storageState` lifecycle (login-as-test)

```
[login-as-test run]                         [prod runs]
PW_NO_TRACE=1                                STORAGE_STATE=/path/state.json
STORAGE_STATE_SAVE=/path/state.json          (pre-loaded cookies/token)
LOGIN_PASSWORD=<from Vault/env>               → newContext({storageState})
plan: navigate→fill(user)→fill(pwd secretRef)  → password is NOT typed
      →click(submit)→assert(url_contains dash)  → trace.zip is clean
→ exit 0 → brain calls browser.saveStorageState → /path/state.json
```

- **Load:** `ensureBrowser()` reads `STORAGE_STATE`; if set and the file is valid → `newContext({…,
  storageState})`. Missing/corrupt file → **fallback to no-state** (don't crash).
- **Save:** `_run_replay`, after a report with `exit_code==0` and when `STORAGE_STATE_SAVE` is set, calls
  `browser.saveStorageState` (creating the parent dir) **before** `traceStop`/`shutdown`.
- **Env propagation:** `STORAGE_STATE` / `STORAGE_STATE_SAVE` / `PW_NO_TRACE` / secret env flow straight
  through agentctl→brain→pw-executor (full `os.environ` inheritance, no allowlist) — **no Go change**.
- **Pluggable auth-adapter** (none|basic|OIDC/Keycloak|storageState-file) and Vault/Helm Secret injection —
  **M9.7** (the `deploy/` chart has no Secret plumbing yet; the seam is `extraEnv` in `cronjob.yaml`).

## §5. Assert / negative testing — semantics

- `browser.expect` returns `{ok}` (does the condition hold). **A step passes iff `ok == expect_ok`**
  (default `expect_ok=true`). The single `expect_ok` flag yields both positive and negative tests.
- **Negative test:** enter something invalid → `assert {condition: "visible", locator: <error>, expect_ok: true}`
  ("error is visible") **or** `{condition: "url_contains", expected: <form>}` ("submit didn't go through, still on the form").
  "Submit blocked" is not a single signal; cross-app-robust ones are an **error element visible** and/or the
  **URL didn't change** (a `disabled` button only works where the app disables it).
- **Exit composition:** a failing `assert` → `passed=False` → the standard `store.record_step`/`failures`
  path (exit 1, quarantine-suppressible). **exit 2 (a11y regression) dominates exit 1** (`replay.py`).
- **Golden capture on an error page:** a golden is captured once per page on first landing; a negative test
  whose page is already in an error state can falsely trip exit 2 against a clean baseline. Mitigation:
  capture the negative plan's baseline **through the same flow** (baseline↔replay symmetry), or mark
  assert pages non-golden. For M9.1 — **documented**, no code added.

## §6. Healing the new verbs (M2 reuse)

`HealingEngine.heal(ctx)` is **verb-agnostic** (works on locators, returns a 1-count locator). Replay for
`fill/type/select`: **probe → (heal if count≠1, applying the step's verb, not click) → act** — a
generalization of the current click path. `assert` **bypasses** probe/heal (a zero count may be the point
of the assertion). `press` without a locator is unhealable (a global key). The explorer catalogs **only
buttons** (`_buttons_from_interactives`), so **hand-authored form plans must supply input `alternatives`**
themselves (testid/label/role=textbox+name); auto-cataloguing inputs is M9.2.

## §7. Invalid-input generator (`brain/validation.py`, sketch)

`invalid_inputs_for(field) -> [{value, expects:{condition, expected, expect_ok}}]` by field type
(`email|number|required|maxlength|pattern`). A pure function, no I/O. `condition/expected/expect_ok` live
in the **frozen step** (under plan_hash), not resolved at replay. A helper assembles `fill`+`assert` pairs.
**A sketch for M9.1**; the full engine (masks, bounds, schema-driven generation) is **M9.2**.
