## Scope

Two independent pw-executor capabilities from the M9 design freeze, delivered in one wave (they share
`pw-executor/src/server.ts`, hence stacked on PR #3):

- **M9.4 / A5 — in-app tab perception** (`role=tab`): tab widgets are now surfaced as interactive.
- **M9.4 / A6 — browser tabs/windows** (multi-page): pw-executor tracks popups/new tabs and can switch
  the active tab.
- **M9.5 / §I — `traceparent` injection** into every browser request: each UI action maps onto the
  customer's end-to-end backend trace (when their services are OTel-instrumented).

## Contract (what is implemented)

### A5 — tab perception
- `browser.interactives` and `browser.setOfMarks` now harvest the selector `… , [role=button], [role=tab]`
  → tab elements enter the map as clickable. Clicking a tab = in-page navigation (the `tabpanel` content
  changes → re-perceive via the existing snapshot).
- **No new brain code** — tabs flow through the existing interactives/site-map.

### A6 — multi-page
- pw-executor keeps `pages: Page[]` (initial + popups/new tabs via `context.on('page')`); `page` is the
  **active** tab. New tools:
  - `browser.tabs` → `{ tabs: [{index, url, title, active}] }` (closed tabs filtered out).
  - `browser.switchTab` `{index}` → makes a tab active (`page = pages[index]` + `bringToFront`).
- **Key property**: switching only re-points the `page` global — **every existing tool**
  (`navigate`/`click`/`fill`/`snapshot`/…) operates on the active tab unchanged.
- Both tools added to `TOOL_METHODS` (both transports: JSON-RPC + MCP).

### M9.5 — `traceparent`
- In `ensureBrowser`, **if `OTEL_EXPORTER_OTLP_ENDPOINT` is set** (tracing on), a `context.route('**/*')`
  is registered that adds a `traceparent` header (from the active OTel span, via
  `otel.ts::currentTraceparent()`) to every outgoing browser request. With no collector it is a **no-op**
  (no route interception, zero overhead).
- This closes §I: UI→frontend→service→Kafka→DB stitches together in the **customer's** Tempo/Jaeger
  (their side). Sentinel stays black-box — **no backend connector** (see DISTRIBUTION §integration).
