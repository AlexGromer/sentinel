# M9.1 Contract — "Form/Login/Validation primitives" (frozen 2026-06-26)

> 🌐 **Русский** (основная версия) · [English](M9.1_CONTRACT.en.md)

Первый под-milestone эпика **M9** (`docs/M9_CONTRACT.md`). Цель — **минимум для живого теста
реальных UI**: формы, поиск, **логин**, многошаговые бизнес-процессы, и **негативное тестирование**
(валидация полей). Закрывает блокер №1 (M9_CONTRACT §A1) тремя частями:

- **GAP-M9-01** — `fill`/`type`/`press`/`select` в `pw-executor` (ввод текста + клавиши + `<select>`).
- **GAP-M9-04** — auth: `storageState`-предусловие + **логин-как-тест**; креды через env, **никогда в артефактах**.
- **GAP-M9-10** — слой ассертов (`assert`-примитив на шаг) + генератор невалидных вводов (**набросок**; полно — M9.2).

Принцип проверки: **offline-first** — `npx tsc --noEmit` (TS) + FakeEx-сьют (Python, без живого
браузера/сети) + `go build ./...`. Живой прогон против реального UI — **отдельный шаг, по явному «go»**.
GoalPlanner / NL-авторинг — **M9.2** (explorer не трогаем); новые виды шагов приходят из **frozen/ручных
планов через replay** (и forward-compat в `act`).

## §1. Новые tools `pw-executor` (wire-схемы)

Регистрируются в **трёх местах** (`pw-executor/src/server.ts`): `dispatchInner` switch, массив
`TOOL_METHODS`, и MCP-`schemas` (zod) — иначе MCP-`registerTool` получит `inputSchema: undefined`.

| Tool | Params | Playwright (pin `^1.61`) | Result |
|------|--------|--------------------------|--------|
| `browser.fill` | `{locator, value? , secretRef?}` | `loc.first().fill(v,{timeout:5000})` | `{filled:true}` |
| `browser.type` | `{locator, text, clear?}` | `clear`→`fill('')`; затем `loc.pressSequentially(text,{timeout})` | `{typed:true}` |
| `browser.press` | `{locator?, key}` | `loc.press(key)` либо `page.keyboard.press(key)` | `{pressed:key}` |
| `browser.select` | `{locator, value}` | `loc.selectOption(value)` | `{selected:[…]}` |
| `browser.expect` | `{locator?, condition, expected?}` | базовые ожидания (`waitFor`/`waitForURL`) — **не бросает** | `{ok:bool, actual?}` |
| `browser.saveStorageState` | `{path}` | `context.storageState({path})` | `{path}` |

Заметки:
- `type` использует **`pressSequentially`** (метод `locator.type()` **deprecated** с PW 1.38).
- `type` **не** очищает поле (append); `fill` очищает автоматически — для логина используем `fill`.
- `press` без `locator` (`page.keyboard`) требует предварительного фокуса; с `locator` — авто-фокус через actionability.
- `select.value` — строка/`{label}`/`{index}`/массив (как `selectOption`); `{selected}` — **только результат**, не часть шага.

### `browser.expect` — условия

`condition ∈ {visible, hidden, enabled, disabled, value_equals, text_contains, count_equals, url_contains}`.
Реализован на **базовых waiting-примитивах `playwright`** (`locator.waitFor({state})`, `page.waitForURL`)
в `try/catch` → `{ok:false}` вместо исключения. **Авто-retry** снимает гонку с пост-submit навигацией
(`@playwright/test` **рассмотрен и отклонён** — GAP-ARCH-001: держим `pw-executor` тонким, без новых зависимостей).
`actual` ограничен счётчиками/URL/булевыми — **никогда** `inputValue()` (чтобы не отразить секрет).

## §2. Схема шага плана (расширение)

Шаг сегодня: `{step_id, action_type, semantic_id, intent, target, locator, alternatives, is_milestone}`.
`action_type` расширяется с `{navigate, click}` до `{navigate, click, fill, type, press, select, assert}`.

| `action_type` | Доп. поля шага |
|---------------|----------------|
| `fill` | `locator`, `value?` **или** `secretRef?`, `alternatives?` |
| `type` | `locator`, `text`, `clear?`, `alternatives?` |
| `press` | `locator?`, `key` |
| `select` | `locator`, `value`, `alternatives?` |
| `assert` | `locator?`, `condition`, `expected?`, `expect_ok?` (default `true`) |

`canonical_plan_hash` хэширует **весь** dict шага → любые новые поля входят в hash. Старые планы
(navigate/click) не меняются. **Replay обязан читать шаг read-only** (резолв `secretRef` и результат
`selectOption` пишутся только в `rec`/результат, не в шаг) — иначе plan_hash «поплывёт».

## §3. Секреты — `secretRef` + tracing-gate (граница безопасности)

**Правило:** креды никогда не попадают в **наши артефакты** (`plan.json`, `llm-transcript.jsonl`,
`heal-report.json`, `rec`, OTel-атрибуты, stderr) **и** в Playwright `trace.zip`.

1. **`secretRef` вместо `value`.** Шаг `fill` для секрета хранит `secretRef` — **имя env-переменной**
   (не секрет), напр. `"LOGIN_PASSWORD"`. Значение резолвится **только внутри `pw-executor`**
   (`process.env[secretRef]`), наружу не возвращается. Brain секрет не видит; plan_hash по `secretRef` безопасен.
   Если env не задана → **throw `"secret <REF> not set"`** (имя не секрет), **никогда** `fill("undefined")`.
2. **Редакция логов/исключений.** Лог: `fill <locator> = <redacted>`. Секрет-`fill` обёрнут в `try/catch`
   → ре-throw `"browser.fill failed (secret redacted)"` (закрывает путь утечки через `message` исключения
   в `executor.py` → stderr/OTel-status).
3. **Tracing-gate (главный фикс утечки в `trace.zip`).** Playwright-трейс пишет **DOM-снапшоты** (значение
   `input.value`, включая `type=password`) **и тело сетевого POST** (форма Keycloak `username`/`password`).
   «Пауза вокруг fill» **не помогает** (секрет уходит на submit-POST и в DOM-снапшот; pause/mask API у
   Playwright нет). Единственный корректный минимум: **`context.tracing.start` включается только если
   `PW_NO_TRACE != "1"`**. Прогон логина-как-теста выставляет `PW_NO_TRACE=1` — его единственный продукт
   `storageState`-файл, трейс не нужен. Прод-прогоны идут с `STORAGE_STATE` (пароль не вводится вовсе) →
   их трейсы чисты by construction. `browser.traceStop` — no-op, если трейс не стартовал. См. **GAP-RISK-010**.
4. **Fail-closed (не только env-дисциплина).** `browser.fill` **бросает**, если трейс активен (`tracingStarted`) —
   **до** чтения `process.env[secretRef]` (секрет даже не загружается). Брейн дополнительно аварийно завершает
   прогон (**exit 3**), если план содержит `secretRef`-шаг при `PW_NO_TRACE != "1"`. Так абсолютный инвариант
   держится кодом, а не только дисциплиной оператора. **Загрузка storageState**: файл парсится в pw-executor
   (не передаётся строкой-путём в `newContext`), поэтому **битый/пустой** файл откатывается к no-state (логируем
   только путь, не байты). Внутренние ожидания `browser.expect` ограничены `page.setDefaultTimeout(5000)` —
   быстрый `{ok:false}` для негативных тестов вместо 30s-дефолта Playwright.

## §4. Auth — `storageState` lifecycle (логин-как-тест)

```
[login-as-test run]                         [prod runs]
PW_NO_TRACE=1                                STORAGE_STATE=/path/state.json
STORAGE_STATE_SAVE=/path/state.json          (предзагруженные cookie/token)
LOGIN_PASSWORD=<из Vault/env>                 → newContext({storageState})
plan: navigate→fill(user)→fill(pwd secretRef)  → пароль НЕ вводится
      →click(submit)→assert(url_contains dash)  → trace.zip чист
→ exit 0 → brain зовёт browser.saveStorageState → /path/state.json
```

- **Загрузка:** `ensureBrowser()` читает `STORAGE_STATE`; если задан и файл валиден → `newContext({…,
  storageState})`. Файл отсутствует/битый → **fallback к no-state** (не падаем).
- **Сохранение:** `_run_replay` после отчёта с `exit_code==0` и при заданном `STORAGE_STATE_SAVE` зовёт
  `browser.saveStorageState` (создаёт родительский каталог) **до** `traceStop`/`shutdown`.
- **Env-проброс:** `STORAGE_STATE` / `STORAGE_STATE_SAVE` / `PW_NO_TRACE` / секрет-env идут насквозь
  agentctl→brain→pw-executor (полное наследование `os.environ`, без allowlist) — **Go-кода не трогаем**.
- **Pluggable auth-adapter** (none|basic|OIDC/Keycloak|storageState-файл) и Vault/Helm Secret-проброс —
  **M9.7** (в чарте `deploy/` Secret-плумбинга пока нет; seam — `extraEnv` в `cronjob.yaml`).

## §5. Assert / негативное тестирование — семантика

- `browser.expect` возвращает `{ok}` (выполняется ли условие). **Шаг проходит, если `ok == expect_ok`**
  (default `expect_ok=true`). Один флаг `expect_ok` даёт и позитивный, и негативный тест.
- **Негативный тест:** ввести невалидное → `assert {condition: "visible", locator: <error>, expect_ok: true}`
  («ошибка видна») **или** `{condition: "url_contains", expected: <форма>}` («submit не прошёл, остались на форме»).
  «Submit заблокирован» — не один сигнал; кросс-аппово надёжны **error-element visible** и/или **url не изменился**
  (`disabled`-кнопка работает лишь там, где приложение её дизейблит).
- **Exit-композиция:** провал `assert` → `passed=False` → штатный `store.record_step`/`failures`
  (exit 1, подавляется карантином). **exit 2 (a11y-регрессия) доминирует над exit 1** (`replay.py`).
- **Golden-капля на странице с ошибкой:** golden снимается раз на страницу при первом заходе; негативный
  тест, чья страница уже в error-состоянии, может ложно дать exit 2 против чистого baseline. Митигировать:
  снимать baseline негативного плана **тем же потоком** (симметрия baseline↔replay), либо пометить
  assert-страницы non-golden. На M9.1 — **документируем**, кода не добавляем.

## §6. Heal новых verb'ов (переиспользование M2)

`HealingEngine.heal(ctx)` **verb-агностичен** (работает с локаторами, возвращает 1-count локатор).
Replay для `fill/type/select`: **probe → (heal при count≠1, применяя verb шага, не click) → act** —
обобщение текущего click-пути. `assert` **минует** probe/heal (нулевой count может быть смыслом ассерта).
`press` без локатора — не лечится (глобальная клавиша). Explorer каталогизирует **только кнопки**
(`_buttons_from_interactives`), поэтому **ручные планы форм обязаны сами задавать `alternatives` инпутов**
(testid/label/role=textbox+name); авто-каталог инпутов — M9.2.

## §7. Генератор невалидных вводов (`brain/validation.py`, набросок)

`invalid_inputs_for(field) -> [{value, expects:{condition, expected, expect_ok}}]` по типу поля
(`email|number|required|maxlength|pattern`). Чистая функция, без I/O. `condition/expected/expect_ok`
живут во **frozen-шаге** (под plan_hash), не резолвятся в replay. Хелпер собирает пары `fill`+`assert`.
**Набросок для M9.1**; полный движок (маски, границы, генерация по схеме поля) — **M9.2**.
