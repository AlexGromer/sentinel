## Scope

ADR-023: driving Sentinel is available via **two paths** — MCP (brain-as-MCP-server, M7) **AND** a thin
**non-MCP HTTP control-API**. M9.3 adds the second path + makes setup-WebUI "live" (phase-2) + ships CI
templates. The chat front (OSS UI) is a separate sub-step on top of the control-API.

## Delivered (Wave B, phase-0 — this PR)
- **setup-WebUI: download** — "Download `run.yaml` / `sentinel.env`" buttons (fully client-side, Blob).
  ⇒ **config generation works fully on GitHub Pages**: generate → download → drop into your local
  instance. No server, no network.

## control-API contract (phase-1 — next sub-step)

A thin **Go HTTP service** (`cmd/control-api`, stdlib `net/http`, like the M8 report-service — no new
deps). This is the "non-MCP" path; the MCP path (M7) coexists (ADR-023).

**Endpoints (v1):**
| Method | Path | Purpose |
|---|---|---|
| GET | `/healthz` | liveness (no auth) → `{status:"ok", version}` |
| GET | `/v1/config-schema` | the RunConfig surface (for the WebUI) — keys/defaults from `runconfig.py`/flags |
| POST | `/v1/runs` | start a run from `{target, runConfig|mode/goal/describe/planner/budgets/auth, env?}` → `{run_id}` (spawns `agentctl run …`, like the orchestrator) |
| GET | `/v1/runs/{id}` | status: `{state: running|done|failed, exit_code, artifact_dir}` |
| GET | `/v1/runs` | list recent runs |

**Security (ADR-032):**
- **Bind to `127.0.0.1` by default** (`CONTROL_API_ADDR`, not `0.0.0.0` without explicit opt-in) — spawning
  runs is a sensitive surface (RCE-class if publicly exposed).
- **Bearer token** (`CONTROL_API_TOKEN`) required for mutations (`POST /v1/runs`); `/healthz` is open.
  With no token in env, mutations return **403** (read-only/health only).
- **CORS allowlist** (`CONTROL_API_CORS_ORIGINS`, e.g. `https://alexgromer.github.io,http://localhost:8088`):
  ⇒ **a Pages page can drive a LOCAL control-API** (localhost is mixed-content-exempt + CORS). Empty by
  default (CORS off; same-origin/bundle only).
- Spawn only via the known `agentctl` path + input validation (no arbitrary commands).

## setup-WebUI phase-2 (with the control-API)
- A **"control-API URL"** field (empty = generation-only; `http://localhost:PORT` = live mode).
- If set + `/healthz` ok → unlock **"▶ Run"** (POST `/v1/runs` with the assembled config) + **status**
  polling (GET `/v1/runs/{id}`). Token is a field (kept only in the tab's memory, never in config/URL).
- **The same file** works: from Pages (→ localhost), from the bundle (`--profile webui`), offline (generation).

## CI templates (GAP-M9-12)
Sentinel = CLI + structured exit codes 0/1/2/3 ⇒ fits any commit-time CI. Ship ready templates **for the
user** (not our GitHub Actions): `Jenkinsfile`, `.gitlab-ci.yml` (replay gate on the exit code). These live
in `docs/ci-templates/` or `examples/` — they do **not** touch `.github/workflows/` (CI-PR #2's territory).
